PART 2

什麼是 AI Agent？

三個關鍵詞定義一切

🎯
自主性
接到目標後，不需要人一步一步指示，自己決定做什麼
🗺️
推理/規劃
把複雜任務拆成步驟，判斷先做什麼、如果失敗怎麼調整
🔧
工具使用
能呼叫 API、搜尋資料、讀寫檔案、操作系統完成任務
= LLM 為核心大腦 + 記憶 + 規劃 + 工具調用

💬 Chatbot（問答型）
你問 → 它答 → 結束。單輪，不會主動採取行動。
🤖 AI Agent（執行型）
你給目標 → 它持續推理、調用工具、自我修正直到完成。

  • ✦ 讀整個 repo 並執行
  • → 真正的執行代理

PART 4 — CLAUDE CODE 深入認識

Claude Code 怎麼讀懂你的專案？

四層設定，優先權由上到下遞減，上層蓋不掉、下層蓋得掉

🏢
企業管理設定 Managed
IT 用 MDM/後台部署，個人與專案設定都蓋不過
🟡
個人專案設定 Local
settings.local.json、CLAUDE.local.md — 自動 gitignore，只在自己機器生效
🟢
專案團隊設定 Project
CLAUDE.md、.claude/settings.json — 進版控，全隊共用
🔵
個人全域設定 User
~/.claude/settings.json、~/.claude/CLAUDE.md — 這台機器上所有專案都套用
📁 CLAUDE.md 支援 monorepo 巢狀載入 + @path 引用其他文件 🧠 Auto Memory 是 Claude 自己寫的筆記，per-project，跟 CLAUDE.md 不同 🔗 業界 AGENTS.md 慣例可用 @AGENTS.md 或 symlink 銜接

PART 4 續

多人協作同一個專案，AI 會搞混嗎？

關鍵事實：Claude Code 本身沒有「使用者身分」的概念——不會因為換人操作就自動變行為，也不記得「這句話是誰說的」。
✅ 規則放對地方
  • ▸團隊規範寫進 CLAUDE.md（進版控），改規則要走 PR 審查，不是口頭跟 AI 講一句就算
  • ▸個人偏好放 settings.local.json／CLAUDE.local.md（自動 gitignore），不會污染別人環境
  • ▸多人平行開發用 git worktree 隔離工作目錄，檔案互不干擾
⚠️ 要注意的地方
  • ▸同一 repo 的所有 worktree 仍共用同一份 Auto Memory，記憶不是依人分開的
  • ▸企業如需強制規範，要靠 Managed 設定，不能只指望大家自律
一句話心法：讓「文件」而不是「誰在講話」決定 AI 的行為。

PRACTICAL GUIDE

開發者現在能用在哪

🔨
程式碼重構
給 agent 一個 repo，說「重構這個模組、補齊 test coverage」，讓它跑
🐛
Debug 協助
把 error log 和 repo 丟給它，讓它自己找問題、提 patch
📝
文件生成
讀懂整個 codebase，自動產出 API 文件、README、規格說明
🧪
自動化測試
根據程式碼自動產生 unit test，跑測試並修正失敗的 case
🔄
CI/CD 整合
把 agent 接進 GitHub Actions，PR 自動 review 或修正小問題
🏗️
快速做 Demo
快速把需求做成可跑的小雛型，用來驗證方向

PART 5 — RISK & PRINCIPLES

導入前要知道的事

⚠️ 常見風險
  • ▸權限過大：agent 可能修改不該改的檔案
  • ▸上下文過長：token 爆炸，成本失控
  • ▸資料外洩：程式碼或敏感資訊上雲
  • ▸Prompt injection：惡意輸入操控 agent
  • ▸過度信任輸出：沒看懂就直接合進去
✅ 導入原則
  • ▸最小權限原則：只給它需要的工具與目錄
  • ▸保留 human-in-the-loop 的覆核點
  • ▸先 PoC → Pilot → Production，逐步擴大
  • ▸設定 token 預算上限與告警機制
  • ▸看懂 agent 的 diff 再 merge，不要盲目接受
核心心態：AI 是加速器，不是替代者。你看不懂的輸出，就是風險所在。

今天現場 demo 的「台股下單系統」，就是照這份確認清單來的：

呈現方式：先備文件與架構鷹架，程式碼現場從零生成
技術棧：C# .NET 8 + SQLite（對應公司內部 .NET + SQL Server）
UI 形式：ASP.NET Core Web API + 簡單網頁前端
資料來源：串接台灣證交所公開資料（真實股價）
功能範圍：查價／下單／委託查詢／取消委託
Demo 時長：30 分鐘以上完整版
這幾個決定，都是動手寫程式之前跟 AI 一來一回討論出來的——這就是 Vibe Coding：人負責定義範圍與取捨，AI 負責提案與執行。
