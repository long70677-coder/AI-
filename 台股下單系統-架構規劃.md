## 系統架構

### 分層架構（N-layer，貼近一般企業 .NET 慣例）
```
StockOrderDemo.sln
├── src/StockOrderDemo.Api/            # ASP.NET Core Web API + wwwroot 靜態網頁前端
├── src/StockOrderDemo.Application/    # Service 層：OrderService、QuoteService（業務邏輯）
├── src/StockOrderDemo.Domain/         # Entity/Enum：Order、OrderSide、OrderType、OrderStatus
├── src/StockOrderDemo.Infrastructure/ # EF Core DbContext(SQLite)、TwseQuoteClient(HttpClient)
└── tests/StockOrderDemo.Tests/        # xUnit：OrderService 業務規則測試
```
這個 5 專案骨架**現場才建立**，用 `dotnet new` 系列指令現場生出來。

### 資料模型（SQLite，EF Core Migration）
`Orders` 資料表：`Id, StockCode, StockName, Side(Buy/Sell), OrderType(Market/Limit), Price(nullable), Quantity, Status(Pending/Filled/Cancelled), CreatedAt, UpdatedAt`

### 台股報價串接
- 資料來源：TWSE 即時報價 snapshot（`https://mis.twse.com.tw/stock/api/getStockInfo.jsp?ex_ch=tse_{代號}.tw`），與 TWSE 官網報價小工具同源，公開可用、免金鑰。
- 限制與因應（架構層級的容錯設計，寫進 CLAUDE.md/架構文件讓 Claude Code 一開始就處理好）：
  - 此 API 有時需要先建立 session（先打一次首頁）才吐得出資料，或非交易時段可能回空值 → `QuoteService` 需設計 fallback：抓不到即時價時，顯示「無法取得即時報價，請確認代號或稍後再試」，不讓下單流程整個掛掉。
  - 範圍限定「上市」股票（`tse_` 前綴），上櫃（`otc_`）列為 out-of-scope 備註，不在本次 demo 範圍。

### 訂單狀態邏輯
- 市價單送出 → 視為立即成交，狀態直接為 `Filled`
- 限價單送出 → 狀態為 `Pending`，可被取消；一旦 `Cancelled`/`Filled` 不可再取消
（無庫存/資金檢查，符合已確認的功能範圍）

## 需求流程

| # | 流程 | 說明 |
|---|------|------|
| 1 | 查價 | 輸入股票代號 → 呼叫 TWSE snapshot API → 顯示代號、名稱、目前成交價、漲跌 |
| 2 | 下單 | 選 買/賣、市價/限價、股數（限價需輸入價格）→ 建立訂單 → 依上述規則設定初始狀態 |
| 3 | 委託查詢 | 列表顯示所有委託單（可依狀態篩選 Pending/Filled/Cancelled） |
| 4 | 取消委託 | 選一筆 `Pending` 委託 → 取消 → 狀態改 `Cancelled`；非 Pending 的單擋掉取消請求並回錯誤訊息 |

## 額外可展示 Vibe Coding 的補充清單

依現場剩餘時間挑著用：
- **Live 除錯**：故意讓一個邊界情況出錯（例如限價單輸入負數股數），丟錯誤訊息給 Claude Code 讓它自己定位並修，順便補一條回歸測試
- **需求異動 demo**：現場加一個新欄位（如訂單備註），觀察它怎麼一次改動 Domain/API/DB Migration/前端，並自動重跑測試
- **Code Review 環節**：用 `/code-review`（或 `code-reviewer` subagent）審查剛生成的 `OrderService`
- **只討論不動手**：問它「如果要加零股下單功能，你會怎麼設計？」只要設計論述，不要它馬上寫 code
- **Subagent 分工**：用 `backend-developer` / `qa-engineer` 兩個 subagent 分別做開發與補測試
- **Git 對照**：每個階段做一次 commit，示範用 `git diff`/log 檢視 Agent 產出
- **反向文件生成**：請它根據剛寫好的程式碼，畫一張 mermaid 循序圖回貼到 docs
- **有無 CLAUDE.md 的對照**：口頭說明同一個提示詞在沒有 CLAUDE.md 時容易猜錯 convention
